# Demo Asset Checklist

## Required Assets

- [ ] Architecture diagram rendered from the Mermaid source.
- [ ] Terminal validation output captured from a clean checkout.
- [ ] Fixture input shown with fake values only.
- [ ] Fixture output shown with sensitive fields redacted.
- [ ] README updated with screenshot links or explicit placeholders.

## Quality Bar

A demo asset should help a skeptical developer understand the repo faster. Do not add decorative images that hide the actual workflow. The best assets show the command, the expected output, and the safety boundary in one view.

## Redaction Review

Before committing any image or GIF, zoom in and inspect every visible terminal line, browser address, path, and account name. Remove private paths, account identifiers, tokens, chat IDs, production workflow IDs, signed URLs, and live finance data.
