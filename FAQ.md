# FAQ

## Is Claude Code OpenRouter Workbench production-ready?

The public repo is ready for docs review, fixture-based onboarding, and safe contribution. Any live integration remains experimental until explicitly documented and validated.

## Can I use this without private services?

Yes. The intended first path uses placeholders, fixtures, and local validation.

## Why not include real credentials or live workflow exports?

Public repos should be safe to clone, inspect, fork, and test. Real credentials, private endpoints, and account-specific logs stay private.

## What should I try first?

Read `QUICKSTART.md`, then run `python validation/validate_repo.py`, then inspect the examples folder.

## What kind of contribution helps most?

A contribution that improves reproducibility: clearer setup docs, better fixtures, stricter validation, or a more honest limitation note.
