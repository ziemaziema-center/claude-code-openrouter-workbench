# Pre-release Checklist

## Documentation

- [ ] Changelog updated.
- [ ] Release notes reviewed.
- [ ] Known limitations reviewed.
- [ ] Demo assets either added or explicitly marked planned.
- [ ] Security docs reviewed.
- [ ] Version tag selected.

## Validation

- [ ] Repo validator passes.
- [ ] Secret scan returns no real values.
- [ ] Public examples use fixtures or placeholders only.
- [ ] No hidden production mutation path exists.

## Release Notes

The first release should be framed as a public preview unless a clean-clone external reviewer has already run the full quickstart successfully.
