# Roadmap

This roadmap is intentionally incremental. The goal is to make Claude Code OpenRouter Workbench easier to understand, safer to clone, and more reproducible before adding broader integrations.

## v0.1 - Public Preview

- Keep docs honest about current limits.
- Validate repo structure and placeholder-only examples.
- Add at least one realistic troubleshooting path.
- Publish with license, security notes, and known limitations.

## v0.2 - Reproducible Demo

- Add a clean-clone fixture demo.
- Capture expected terminal output.
- Add rendered architecture screenshot.
- Convert one manual checklist into a script.

## v0.3 - Maintainer Feedback

- Ask early reviewers to run the quickstart.
- Convert confusing setup steps into issues.
- Add edge cases discovered during review.
- Improve docs based on actual questions, not imagined scale.

## v1.0 - Stable Public Contract

- Clear file structure.
- Passing CI.
- Reproducible quickstart.
- No unresolved public/private boundary ambiguity.

## Not on the Roadmap Yet

- Hosted SaaS.
- Fully autonomous execution.
- Live production integrations enabled by default.
- Claims of adoption before real users exist.
